DBファイルの基本フォーマット
// fish-db-example.js (function() { window.__FISH_DB_QUEUE = window.__FISH_DB_QUEUE || []; window.__FISH_DB_QUEUE.push({ meta: { id: "my-db-v1", // 一意のID（英数字+ハイフン） name: "自店カスタムDB", version: "1.0.0" // セマンティックバージョン }, entries: { "my_fish_001": { group: "luxury", // CATEGORIES のキー name: "本マグロ大トロ", price: 15000, // 円/kg（正の整数） loss: 3, // 鮮度劣化日数 p_loss: 45, // 廃棄率% (100-歩留まり%) decay: 20, // 歩留まり実績% guide: 2, // 推奨保存日数 source: "自店調査2024" } } }); })();
group（カテゴリ）の一覧
バリデーションルール（自動チェック）
  • price: 1以上の整数（0以下はエラー）
  • p_loss（廃棄率）: 0〜100の整数
  • guide（保存日数）: 0以上の整数
  • group: CATEGORIES に含まれるキー
  • name: 空白・未定義はエラー
  • バリデーション失敗のエントリはスキップ（警告表示）
バージョン競合について

既に読み込み済みのモジュールと同じIDで、バージョンが異なるDBを読み込もうとすると警告が表示されます。意図的な更新の場合は「続行」を選択してください。

高級寿司ネタ特化モード（BOKUMO推奨）

「標準DBを除外」ボタンで標準DBをメモリからクリアし、カスタムDBだけを読み込むことで、選択肢をスッキリさせた特化モードで運用できます。ページリロードで標準DBに戻ります。
